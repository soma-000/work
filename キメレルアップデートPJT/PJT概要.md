# ゴール
- キメレルの成果を向上させるための、アップデートの実施
## 背景
- キメレルの競合が増えている
- 成果を出せないことによって炎上する案件が増加
# ステップ
1. 課題からアップデートする考え方
	1. 成果が出ている5KWと出ていない5KWの計10KWに対して、競合調査を実施する
	2. 野中さんとディスカッションをする
	3. 1,2を繰り返して、課題及び施策をブラッシュアップ/整理する
2. 理想状態と今をつなぐ考え方
	1. 計4つの課題の理想状態を考える
	2. 理想状態が実現されている状態及びサイト例を調査する
	3. 野中さんとディスカッションする
	4. 1,2,3を繰り返して、理想状態等をブラッシュアップ/整理する
3. キメレルのVPを考えるやり方
	1. キメレルの競合調査等を繰り返して、出てきたキメレルのVPを調査する
	2. 野中さんとディスカッションする
	3. 1,2を繰り返して、ブラッシュアップする
4. 施策を考える？
## 備考
- 理想状態を考える上で詰まったら、他業界の事例を見て、どうやって機能をアップデートしたのかを考える
	- paypay等
- 定性的にアップデート案を考える（定量的な考え方は、CROの文脈で）
# 課題
1. ランキング（おすすめ3選）の情報が薄い
   ・比較表内の項目の再掲が中心で、新規情報があまりない
   ・この項目だけ見てもサービスのイメージが湧きにくく、送客につながらなさそう
2. ファクトにこだわりすぎて、サイトに人間味がない/機械的に感じる
   ・競合は口コミの導入等をしているのに対し、キメレルは無機質すぎるので、ユーザーに情報が入りくい
3. 送客の押しが弱い
   ・ポップアップ/CTA等がほとんどなく、CVポイントが少ない
4. AIO対策が弱い
