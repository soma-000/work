# キメレル アップデートプロジェクト構成案

## 1. プロジェクトの背景と目的

競合サイトの台頭や成果未達案件の増加を受け、キメレルのフォーマット自体を本質的にアップデートし、**「クライアント企業への送客（1社への集中）」**を最大化することを目的とする。

## 2. 課題抽出のロジック（なぜこの4点なのか）

本プロジェクトにおける4大課題は、個人の主観ではなく、以下のデータおよび調査工程を経て絞り込まれたものである。

1. **10KWの徹底比較調査**: 成果が出ている5KWと出ていない5KWの計10KWを対象に競合調査を実施。
    
2. **改善余地のカテゴライズ**: 調査で得られた膨大な改善ポイントを、「実現可能性」と「緊急性・インパクト」の2軸で整理。
    
3. **共通項の抽出**: どのキーワードにおいても共通して「ユーザーの離脱ポイント」や「競合が勝っている要素」となっていた以下の4点に集約。

## 3. キメレル アップデート検証表

各課題に対し、ユーザーが読み終わった瞬間の「読後感」を理想状態として定義。クライアント企業への確信を生み、他社への離脱（検索画面への戻り）を阻止するロジックで構成している。

|**課題**|**課題の説明**|**理想状態：読後感の定義**|**解決に必要な要素（ブレイクダウン）**|**検証基準**|
|---|---|---|---|---|
|**①ランキング（おすすめTOP3）の情報が薄い**|・比較表の再掲中心で新規情報が乏しい<br><br>  <br><br>・サービスのイメージが湧かず送客しにくい|**「クライアント企業の価値が立体的に理解できた。これなら安心して（クライアント企業へ）決めて大丈夫だ。他を調べる必要はない」**|1. **比較表を超えた「独自ベネフィット」の提示**：表の再掲ではなく、メリット・デメリット・懸念点まで踏み込んだ深い解説<br><br>  <br><br>2. **利用実感を伴う「具体性の付与」**：スマホ実機の操作感や詳細な会社概要、導入事例の可視化<br><br>  <br><br>3. **意思決定を完結させる客観的証明**：口コミやよくある質問による、その場での疑問解消|・「もっと詳しい情報を探そう」という離脱意欲が消滅しているか？<br><br>  <br><br>・クライアント企業のサービス利用シーンが具体的にイメージできているか？|
|**②ファクトにこだわりすぎて、サイトに人間味がない/機械的に感じる**|・競合に比べ無機質で、ユーザーに情報が入りにくい|**「ぶっきらぼうな情報ではなく、信頼できる人が私のことを想って（クライアント企業を）勧めてくれている暖かさを感じる」**|1. **専門家の「顔」が見える推奨**：編集部や監修者の顔・コメントによる、責任ある情報の演出<br><br>  <br><br>2. **ユーザーに寄り添う「双方向性」**：自分に合うか判断できる診断やターゲットの明示<br><br>  <br><br>3. **「人の気配」を感じるコンテンツ**：実際のアンケート結果や口コミなど、第三者の生々しい声を配置|・情報の「提示」ではなく、信頼できる人からの「推奨」として受け取られているか？<br><br>  <br><br>・クライアント企業の紹介文に制作側の「意思」が見えるか？|

## 4. 施策の対応一覧

- **送客機能**: ポップアップ、診断コンテンツ、各種レポート/資料DL、追従型ボタン

### 2. これからやらないといけないこと（今後のタスク）

GW明けの実装開始に向けて、意思決定のプロセスと資料の形式を整えるフェーズに入ります。

#### ① 課題選定の「裏付けデータ」の整理

- **内容**: 「なぜこの4つの課題なのか」を説明するため、10KW調査の中から「どのKWを見てもこの不満を感じた」という具体的な気づきを証跡として残す。
    
- **目的**: 外部からの「この課題で合っているのか？」という疑問に即答できるようにする。
    

#### ② 施策の決定プロセス（段取り）の確立

- **内容**: どのチーム（プロダクト・集客など）と合意を取り、誰が最終決定を下すのかというフローを確定させる。
    
- **目的**: GW明けにスムーズに改修作業へ移行できるようにする。
    

#### ③ 優先順位（ROI）の評価

- **内容**: 洗い出した施策に対し、「成果へのインパクト」と「実装の工数」の2軸でマトリクス化し、着手順を決める。
    
- **目的**: 最も緊急性が高く、バリューが上がるポイントから着手する。
    

#### ④ キメレルのVP（独自価値）の再定義

- **内容**: 競合調査を経て見えてきた「キメレルだからこそ提供できる価値」を言語化し、施策に反映させる。
    
- **目的**: どの案件に横展開しても機能する「強いフォーマット」を完成させる。
    

#### ⑤ スプレッドシート（共有資料）の構造アップデート

- **内容**: 現在のスプシを「説明用資料」として機能するよう、以下のシート構成に整える。
    
    - **Dashboard**: PJT全体像・4大課題・マイルストーン
        
    - **Research**: 10KW調査の生データと気づきのメモ
        
    - **Logics**: 課題から読後感へのつながり（検証基準）
        
    - **Measures**: 優先順位付き施策リスト
        
    - **Schedule**: SAS（ガントチャート）
