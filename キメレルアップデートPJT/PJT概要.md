# PJTについて
## ゴール
- キメレルの成果を向上させるための、アップデートの実施
## 背景
- キメレルの競合が増えている
- 成果を出せないことによって炎上する案件が増加
---
# スケジュールについて
## ステップ一覧表

| **ステップ**               | **ステップの説明**                                          | **次のステップに進むための条件**                                       | **成果物**                                  |
| ---------------------- | ---------------------------------------------------- | -------------------------------------------------------- | ---------------------------------------- |
| **1. 競合調査（10KW調査）**    | 成果が出ている5KWと出ていない5KW（計10KW）を調査し、競合との差異を分析。            | ・10KW分の調査データが揃っていること。<br><br>・各KWごとの「改善の気づき」が言語化されていること。 | ・**競合比較調査シート**<br><br>・**KW別改善点/気づきリスト** |
| **2. 課題の抽出とカテゴライズ**    | 調査で得られた改善点を整理し、キメレル全体に共通する課題を洗い出す。                   | ・課題を整理し、カテゴライズできていること。                                   | ・**課題リスト**                               |
| **3. 課題の絞り込み**         | 抽出した課題を特定の軸で評価し、絞り込む                                 | ・各課題の選定理由にデータ（KW調査結果）との紐付けができていること。                      | ・**課題の選定基準**                             |
| **4. キメレルのVPの策定**      | 理想状態を考える上で、キメレルの強みを活かすためにVPを策定すrう                    | ・キメレルのVPが策定できていること                                       | ・**キメレルのVP**                             |
| **5. 理想状態の言語化**        | 課題が解決された際、ユーザーがサイトを読み終わった瞬間にどのような心理状態（読後感）になるかを定義する。 | ・各課題に対し、理想状態が言語化し、どの要素があればその理想状態が満たされるのかガ明確であること。        | ・**理想状態の言語化**<br><br>・**各理想状態の必要要素**     |
| **6. 施策案の棚卸しとROI評価**   | 理想状態を実現するための施策を整理し、選定できている。                          | ・施策をある軸を基に判定できていること。                                     | ・**施策の選定基準**<br><br>・**選定施策**            |
| **7. チーム・関係部署合意**      | 未定                                                   | 未定                                                       | 未定                                       |
| **8. 谷口さんへの最終プレゼン・承認** | 未定                                                   | 未定                                                       | 未定                                       |
